- `bundle` 类
  | 构建工具 | 描述 |
  | --- | --- |
  | `gulp` | `gulp`是一个自动化构建工具，主要用来设定程序自动处理静态资源的工作。简单的说，`gulp`就是用来打包项目的。`gulp`是一个基于任务的工具，`gulp`规定不管做什么功能，都用统一的接口管理，必须去注册一个任务，然后去执行这个任务，在任务代码中去做想做的功能。 |
  | `grunt` | 与`gulp`类似，也是一个用来定义并运行任务的工具。`grunt`和`gulp`最大的不同就在于`grunt`使用配置对象来声明任务的运行方式，而`gulp`则使用`javaScript`的函数来定义任务。 |
  | `webpack` | 本质上，`webpack`是一个用于现代`javaScript`应用程序的静态模块打包工具，它是基于入口的，当`webpack`处理应用程序时，它会递归的解析入口所需要加载的所有资源文件，然后用不同的`loader`处理不同的文件，采用`plugin`扩展功能，最终在内部构建一个依赖图，此依赖图对应映射到项目所需的每个模块，并生成一个或多个`bundle`，适用于大型复杂的前端站点构建。 |
  | `parcel` | `parcel`是`web`应用打包工具，适用于经验不同的开发者。它利用多核处理提供了极快的速度，并不需要任何配置。 |
  | `rollup` | `rollup`是一个`javaScript`模块打包器，可以将小块代码编译成大块复杂的代码，例如`library`基础库或应用程序。 |
  | `esbuild` | `esbuild`使用`golang`进行开发，基于`ESM`，是下一代构建工具，并且它的打包速度极快，相较于其他`web`构建工具来说，速度大约是其`10~100`倍。 |

- `no bundle` 类
  | 构建工具 | 描述 |
  | --- | --- |
  | `snowpack` | `snowpack`是为现代浏览器设计的一个超级快的前端构建工具，`snowpack`利用`javaScript`的本地模块系统(即：`ESM`)来避免不必要的(转换)工作，并无论工程增长多大时保持运行 快速。 |
  | `vite` | 在`dev`模式下，`vite`的预构建、文件编译使用的是`esbuild`；
  在`prod`模式下，`vite`采用`rollup`进行打包，这是因为`esbuild`一些针对构建应用的重要功能仍然在持续开发中---特别是 **代码分割 **和 **CSS处理 **方面。 |
