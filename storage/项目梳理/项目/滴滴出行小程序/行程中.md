订单流转，就是一个接口轮训去反复调用接口，隔一段时间调用一次，也会有 websocket <a name="H1MZd"></a>

## 订单状态

- 状态 status
  - 0：默认值
  - 1：待出发
  - 2：已关闭
  - 3：已完成
  - 4：机型中
  - 5：待支付
  - 6：已取消
  - 7：待抢单
- 子状态 sub\_status
  - 0：默认值
  - 2001：客服关闭
  - 2002：改派关闭
  - 2003：未能完成服务关闭
  - 4001：等待接驾
  - 4002：司机迟到
  - 4003：司机到达
  - 4004：乘客迟到
  - 4006：计费中
  - 5001：正常订单待支付
  - 5002：乘客取消订单待支付
  - 5003：司机取消订单待支付
  - 6001：乘客无责取消行程
  - 7003：指派预约单，播单前状态
  - 7004：指派预约单，播单中
- 场景类型 combo\_type
  - 0：普通
  - 1：包车
  - 2：接机
  - 3：送机
  - 4：拼车
  - 302：跨区域拼车
  - 303：区域一口价
  - 308：深港一口价
  - 314：特价车
